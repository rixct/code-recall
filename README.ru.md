# CodeRecall

> Anki + LeetCode внутри Obsidian — интервальное повторение кода с автопроверкой.

**English version → [README.md](README.md)**

---

## Что это

CodeRecall превращает блоки кода в твоих заметках [Obsidian](https://obsidian.md)
в карточки для интервального повторения. Ты прячешь часть кода и при повторении
дописываешь её по памяти. Вместо «показал ответ → сам оценил» плагин **запускает
твой код** на тест-кейсах и оценивает автоматически — карточка считается
вспомненной, только если код действительно работает.

Карточки без тестов (или без языка) проверяются **сравнением твоего текста со
скрытым ответом**, так что подходят и для обычных сниппетов, и для текста.

## Карточка

`{{c1::…}}` — cloze-делеция, тот фрагмент, что скрывается при повторении.

    ```coderecall
    lang: python
    name: Two Sum
    code: |
      def two_sum(nums, target):
          seen = {}
          {{c1::for i, n in enumerate(nums):
              if target - n in seen:
                  return [seen[target - n], i]
              seen[n] = i}}
    tests:
      - in: "[[2, 7, 11, 15], 9]"
        out: "[0, 1]"
      - in: "[[3, 2, 4], 6]"
        out: "[1, 2]"
    ```

При повторении скрытая часть заменяется полем ввода. Ты пишешь ответ, жмёшь
**Запустить и проверить**, плагин прогоняет тесты в песочнице и показывает
pass/fail.

## Формат карточки

Карточка — один fenced-блок с info-строкой `coderecall`; тело — YAML:

| Поле | Обязательно | Значение |
|---|---|---|
| `code` | да | Твой код с одной или несколькими cloze-делециями `{{cN::…}}`. |
| `lang` | нет | Язык (`python`, `javascript`, `c++`, `java`). **Не указывай его** для текстовой карточки — она проверяется сравнением, а не запуском. |
| `tests` | нет | Список пар `{ in, out }`. **Без тестов карточка оценивается сравнением твоего ответа со скрытым текстом.** |
| `name` | нет | Подпись в режиме повторения. |
| `entry` | нет | Entry-функция для тестов (определяется автоматически, если не задана). |
| `mode` | нет | `call` (вызвать entry-функцию, сравнить возврат) или `stdio` (подать `in` в stdin, сравнить stdout). По умолчанию: `call` для JS/Python, `stdio` для C++/Java. |
| `id` | нет | Переопределение стабильного id (по умолчанию — хеш содержимого). |

Вместо встроенного `code: |` можно разделить **заголовок / код / тесты** строками
`---` — тогда код берётся дословно, с отступами и табами:

    ```coderecall
    name: Столица Франции
    ---
    Столица Франции — {{c1::Париж}}.
    ```

У этой карточки нет `lang` и нет `tests`, поэтому она оценивается по тексту: твой
ответ сравнивается со скрытым содержимым **без учёта пробелов** (`print(x + y)` и
`print(x+y)` оба подходят под скрытое `print(x + y)`).

**Синтаксис cloze:** `{{cN::скрытое}}` прячет `скрытое` как группу *N*; закрывает
первый неэкранированный `}}`. Чтобы вставить литеральные `{{` или `}}` внутрь
скрытого текста, экранируй как `\{{` / `\}}` — любой другой бэкслеш (`\n`, `\\`)
остаётся как есть.

## Языки

| Язык | Раннер | Примечания |
|---|---|---|
| JavaScript | Web Worker | офлайн, режим `call` |
| Python | Pyodide (WASM) | скачивается один раз при первом запуске, `call` |
| C++ | локальный g++/clang++ | полный STL, режим `stdio`; фолбэк на JSCPP (без STL), если нет компилятора |
| Java | локальный JDK 11+ | режим `stdio`, single-file запуск |

C++/Java запускают реальные программы на твоей машине через локальные тулчейны,
поэтому плагин **только для десктопа** — включай тумблер **Native execution** в
настройках и открывай только доверенные заметки. Подсветка работает для них и для
любого языка, который знает Obsidian (TypeScript, Go, Rust, …), даже если
запускать его нельзя.

## Повторение и оценки

Запусти **Повторить текущую заметку** (иконка «мозг» на панели или команда). По
каждой карточке:

- **Запустить и проверить / Проверить** — оценивает ответ (прогоняет тесты или
  сравнивает текст для карточек без тестов) и показывает результат. Расписание
  при этом не меняется.
- **Показать ответ** — показывает эталон и переключает на ручную оценку.
- После прогона появляется авто-вердикт — **Продолжить (всё верно / частично /
  неверно)** ставит quality 5 / 3 / 1, кнопки **Изменить** переопределяют его.
  При показе ответа или самооценке выбираешь **Снова / Трудно / Хорошо / Легко**.

Что каждая оценка делает с расписанием SM-2 (хранится в `data.json`):

| Кнопка | Quality | Эффект |
|---|---|---|
| Снова | 1 | Провал: серия сбрасывается, показ снова через 1 день |
| Трудно | 3 | Зачёт, ease −0.14 (интервалы растут медленнее) |
| Хорошо | 4 | Зачёт, ease без изменений |
| Легко | 5 | Зачёт, ease +0.10 (интервалы растут быстрее) |

Серия успешных повторений даёт следующий показ через **1 день → 6 дней →
предыдущий × ease** (ease стартует с 2.5, минимум 1.3).

## Язык интерфейса

Интерфейс CodeRecall доступен на **русском** и **английском**. Он автоматически
следует за языком Obsidian; принудительно выбрать язык можно в
**Settings → CodeRecall → Язык** (Авто / English / Русский).

## Установка

Пока не в Community Plugins. Чтобы запустить вручную, собери (см. ниже) и скопируй
`manifest.json`, `main.js` и `styles.css` в
`<vault>/.obsidian/plugins/coderecall/`, затем включи плагин. Готовую колоду для
пробы смотри в [examples/sample-deck.md](examples/sample-deck.md).

## Разработка

```bash
npm install
npm run dev     # сборка в watch-режиме
npm test        # юнит-тесты (vitest)
npm run build   # проверка типов + production-бандл
```
