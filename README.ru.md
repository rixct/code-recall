# CodeRecall

> Anki + LeetCode внутри Obsidian. Активное вспоминание кода с автопроверкой.

**English version → [README.md](README.md)**

---

## Что это

CodeRecall — плагин для [Obsidian](https://obsidian.md), который превращает
блоки кода в твоих заметках в карточки для интервального повторения (spaced
repetition) — но не в обычные «показал ответ → сам оценил себя», а с
**автоматической проверкой**.

Ты прячешь часть кода (cloze-делеция), а при повторении:

1. дописываешь скрытый фрагмент по памяти;
2. плагин **реально выполняет** твою версию (та же логика, что в Execute Code);
3. сверяет вывод с эталонными тест-кейсами;
4. по результату (прошёл / не прошёл) планировщик SM-2 назначает следующий показ.

Это автограйдинг активного вспоминания: карточка считается вспомненной, только
если твой код действительно работает, а не потому что ты сам себе поставил
«помню».

## Зачем (в чём дыра)

- **Spaced Repetition** плагин — это только текстовые карточки. Нет выполнения,
  нет автопроверки кода. Оценка честности — на совести пользователя.
- **Execute Code** плагин — умеет запускать код прямо в заметке, но никак не
  связан с памятью и повторением.

Никто не соединил оба направления. CodeRecall закрывает этот разрыв: **память +
исполнение + автопроверка** в одном месте.

Аудитория — не только сисадмины (как у Execute Code), а все, кто учит
алгоритмы, синтаксис языков, API, паттерны: студенты, джуны, люди на собесах,
switchers.

## Как выглядит карточка

Пример: карточка со скрытым телом функции. `{{c1::...}}` — cloze-делеция
(скрываемый фрагмент).

    ```coderecall
    lang: python
    name: Two Sum
    code: |
      def two_sum(nums, target):
          seen = {}
          {{c1::for i, n in enumerate(nums):
              if target - n in seen:
                  return [seen[target - n], i]
              seen[n] = i}}
    tests:
      - in: "[[2, 7, 11, 15], 9]"
        out: "[0, 1]"
      - in: "[[3, 2, 4], 6]"
        out: "[1, 2]"

При повторении скрытая часть заменяется на поле ввода. Ты пишешь код → жмёшь
«Проверить» → плагин прогоняет тест-кейсы в песочнице → показывает
pass/fail и обновляет расписание.

### Формат карточки

Карточка — это один fenced-блок с info-строкой `coderecall`. Тело — YAML:

- **`lang`** (обязательно) — идентификатор языка, напр. `python`, `javascript`.
- **`code`** (обязательно) — YAML-блок-скаляр (`code: |`) с твоим кодом и одной
  или несколькими cloze-делециями.
- **`tests`** (опционально) — список пар `{ in, out }`.
- **`name`** / **`entry`** / **`id`** (опционально) — подпись, переопределение
  entry-функции, переопределение стабильного id.
- **`mode`** (опционально) — `call` (по умолчанию для JS/Python: вызвать
  entry-функцию с JSON-аргументами, сравнить возврат) или `stdio` (по умолчанию
  для C++/Java: подать `in` в stdin, сравнить stdout).

Рекомендуемая раскладка разделяет **заголовок / код / тесты** строками `---` —
тогда код берётся дословно (отступы и табы сохраняются); встроенный `code: |`
тоже работает.

Языки:

- **JavaScript** — Web Worker, офлайн (режим `call`).
- **Python** — Pyodide/WASM, скачивается один раз (режим `call`).
- **C++** — компилируется локальным **g++/clang++**, полный STL (режим `stdio`).
  Без компилятора — фолбэк на встроенный JSCPP (без STL).
- **Java** — через локальный **JDK 11+** (single-file режим, `stdio`).

C++/Java используют нативные тулчейны через Node, поэтому плагин **только для
десктопа**, и их запуск исполняет реальные программы на твоей машине — включается
тумблером **Native execution** в настройках; открывай только доверенные заметки.

Синтаксис cloze: `{{cN::скрытое}}` прячет `скрытое` как группу *N*. Закрывает
первый неэкранированный `}}`. Чтобы вставить литеральные `{{` или `}}` внутрь
скрытого кода (f-строка, dict-литерал в конце строки) — экранируй как `\{{` /
`\}}`; любой другой бэкслеш (`\n`, `\\`) остаётся как есть. Вложенные, пустые и
незакрытые cloze считаются ошибкой.

В режиме чтения и Live Preview блок `coderecall` рендерится компактной карточкой
с подсветкой секции кода (по `lang`). Подсветку можно выключить в
**Settings → CodeRecall**.

## Статус

**MVP.** Полный цикл работает: парсинг карточек → прячем код → вписываешь ответ →
запуск в песочнице → автопроверка по тестам → расписание SM-2. Что дальше — в
[ROADMAP.md](ROADMAP.md) (очередь «на сегодня» по всему хранилищу, статистика).

## Поддерживаемые языки

| Язык | Раннер | Примечания |
|---|---|---|
| JavaScript | Web Worker | офлайн, режим `call` |
| Python | Pyodide (WASM) | скачивается один раз при первом запуске, `call` |
| C++ | локальный g++/clang++ | полный STL, режим `stdio`; фолбэк на JSCPP (без STL) если нет компилятора |
| Java | локальный JDK 11+ | режим `stdio`, single-file запуск |

Подсветка работает для них и для любого языка, который знает Obsidian
(TypeScript, Go, Rust, …), даже если запускать его пока нельзя.

## Повторение и оценки

Запусти **Review current note** (иконка в панели или команда). Каждая карточка
показывает код со скрытой частью и поле на каждый cloze.

- **Run & check** — подставляет твой ответ, прогоняет все тесты в песочнице,
  показывает pass/fail по каждому (expected vs got). Это только *проверка* —
  расписание не меняет.
- **Reveal answer** — показывает эталон и переключает на ручную оценку.
- После прогона появляется авто-вердикт: **Continue (all passed / partial /
  failed)** ставит quality 5 / 3 / 1; **Override: Again** (1) или **Override:
  Good** (4) — переопределить.
- После Reveal или если у карточки нет тестов — оцениваешь сам: **Again / Hard /
  Good / Easy**.

Что каждая оценка делает с расписанием SM-2 (хранится в `data.json`):

| Кнопка | Quality | Эффект |
|---|---|---|
| Again | 1 | Провал: серия сбрасывается, показ снова через **1 день**, счётчик срывов +1 |
| Hard | 3 | Зачёт, но **ease −0.14** → дальше интервалы растут медленнее |
| Good | 4 | Зачёт, ease без изменений |
| Easy | 5 | Зачёт, **ease +0.10** → дальше интервалы растут быстрее |

Серия успешных повторений даёт следующий показ через **1 день → 6 дней →
предыдущий × ease** (ease стартует с 2.5, минимум 1.3). «Continue» просто
применяет quality из авто-вердикта, так что вручную расписание трогать обычно
не нужно.

## Безопасность

Выполнение чужого/своего кода — это риск. Подход к песочнице описан в
[ROADMAP.md](ROADMAP.md#безопасность-и-песочница). Кратко: изоляция обязательна,
запуск произвольного кода без песочницы делать не будем.

## Установка

Пока не опубликовано. После MVP — через BRAT или Community Plugins.

## Разработка

```bash
npm install
npm run dev     # сборка в watch-режиме
npm test        # юнит-тесты парсера (vitest)
npm run build   # проверка типов + production-бандл
```

Затем скопируй/симлинкни `manifest.json`, `main.js` и `styles.css` в
`<vault>/.obsidian/plugins/coderecall/` и включи плагин.

## Как пользоваться

1. Скопируй [examples/sample-deck.md](examples/sample-deck.md) в своё хранилище
   и открой (или напиши свой блок `coderecall`).
2. Выполни **«CodeRecall: Review current note»** — иконка «мозг» в левой панели
   или палитра команд.
3. По каждой карточке: впиши скрытый код в поле, нажми **Run & check**. Увидишь
   pass/fail по каждому тесту с expected/got, затем выбери оценку (или прими
   авто-вердикт) — SM-2 назначит следующий показ.

Команды: **Review current note** и **Scan active note for cards** (сухой прогон:
просто показывает, сколько карточек распарсилось). Состояние повторений хранится
в `data.json` плагина. Первая JS-карточка работает без интернета; Python-карточка
скачивает Pyodide при первом запуске.
